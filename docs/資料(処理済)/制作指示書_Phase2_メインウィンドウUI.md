# 制作指示書：第2フェーズ - メインウィンドウUI構築

## この指示書について

C#開発（司令塔）からClaudeCode（実行者）への制作指示です。
第1フェーズ（骨格とデータ層）が完了したことを前提に、メインウィンドウのUIを構築します。

**まず本指示書を読み、把握レポートを出力してください。コード生成はまだ行わないでください。**

---

## 第2フェーズの範囲

### 作るもの
- MainForm.cs のUI構築（プロファイル一覧と操作ボタン）
- アプリ起動時にプロファイル一覧を読み込んで表示する処理
- 各ボタンのイベントハンドラ（骨格のみ）
- プロファイル削除機能（確認ダイアログ付き）

### 作らないもの（次フェーズ以降）
- プロファイル編集画面（EditForm）
- 「現在を拾う」機能
- プロファイル実行機能（アプリ起動・配置）
- Win32 API呼び出し

## MainForm.cs のUI設計

### ウィンドウ基本設定
- タイトル：「StartForm」
- サイズ：幅600 × 高さ450
- StartPosition：CenterScreen
- MinimumSize：幅500 × 高さ350（これ以下に縮小できない）
- FormBorderStyle：Sizable（リサイズ可能）

### 各コントロールの仕様

#### プロファイル一覧（ListBox）
- 変数名：lstProfiles
- Dock：なし（Anchorで制御）
- Anchor：Top, Left, Right, Bottom（ウィンドウリサイズに追従）
- 位置・サイズの目安：左右マージン20px、上マージン20px、下はボタン行の上まで
- SelectionMode：One（単一選択）
- Font：メイリオ, 10pt（日本語が読みやすいフォント）
- 表示内容：プロファイル名（ProfileName）を表示

#### ボタン行
フォーム下部に横並びで配置する。Anchor は Bottom, Left で固定。

| ボタン | 変数名 | テキスト | 幅 | 備考 |
|--------|--------|---------|-----|------|
| 実行 | btnExecute | 実行 | 100 | プロファイル選択時のみ有効 |
| 新規作成 | btnNew | 新規作成 | 100 | 常に有効 |
| 編集 | btnEdit | 編集 | 100 | プロファイル選択時のみ有効 |
| 削除 | btnDelete | 削除 | 100 | プロファイル選択時のみ有効 |

- ボタンの高さ：35px
- ボタン間のマージン：10px
- 下マージン：20px、左マージン：20px
- Font：メイリオ, 9pt

## 処理の実装

### 1. フォーム初期化時（MainForm コンストラクタまたは Load イベント）

```
処理：
  1. ProfileService.LoadAll() でプロファイル一覧を取得
  2. ListBox にプロファイル名を表示
  3. ボタンの有効/無効を更新（初期状態：実行・編集・削除は無効）
```

### 2. プロファイル一覧の選択変更時（lstProfiles.SelectedIndexChanged）

```
処理：
  - 何か選択されている → 実行・編集・削除ボタンを有効にする
  - 何も選択されていない → 実行・編集・削除ボタンを無効にする
```

### 3. 実行ボタン（btnExecute.Click）

```
処理：
  - 第2フェーズでは未実装
  - MessageBox.Show("プロファイルを実行します（未実装）", "StartForm") を表示
```

### 4. 新規作成ボタン（btnNew.Click）

```
処理：
  - 第2フェーズでは未実装
  - MessageBox.Show("新規作成画面を開きます（未実装）", "StartForm") を表示
```

### 5. 編集ボタン（btnEdit.Click）

```
処理：
  - 第2フェーズでは未実装
  - MessageBox.Show("編集画面を開きます（未実装）", "StartForm") を表示
```

### 6. 削除ボタン（btnDelete.Click）

```
処理：
  1. 確認ダイアログを表示：
     "「{プロファイル名}」を削除しますか？"
     ボタン：はい / いいえ
  2. 「はい」の場合：
     - ProfileService.Delete(profileName) で削除
     - プロファイル一覧を再読み込みして表示を更新
  3. 「いいえ」の場合：何もしない
```

### 7. プロファイル一覧の再読み込みメソッド

```
private void RefreshProfileList()
  1. ProfileService.LoadAll() でプロファイル一覧を再取得
  2. ListBox をクリアして再表示
  3. ボタンの有効/無効を更新
```

このメソッドは、初期化時・削除後・将来の編集画面から戻った時に共通で使う。

## 完了条件

1. `dotnet build` がエラーなく通ること
2. アプリを起動すると、プロファイル一覧（ListBox）と4つのボタンが表示されること
3. プロファイルが存在しない状態では、一覧は空、実行・編集・削除ボタンは無効であること
4. 削除ボタンは確認ダイアログを表示すること
5. 実行・新規作成・編集ボタンはMessageBoxで「未実装」と表示すること
