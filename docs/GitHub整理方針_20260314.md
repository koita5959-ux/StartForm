# GitHub整理方針 2026-03-14

## 対象

`C:\Users\PC\Desktop\StartForm開発\StartForm`

## いまの分類

### 1. ソースコード本体

- `Forms/`
- `Helpers/`
- `Models/`
- `Services/`
- `Program.cs`
- `StartForm.csproj`
- `I.ico`

これはアプリの実体です。GitHub の中心になります。

### 2. ビルド・配布設定

- `setup.iss`

インストーラー生成に必要です。GitHub に残す価値があります。

### 3. 利用者向け文書

- `ReadMe.txt`
- `ご利用にあたって.txt`

配布や説明に使う文書です。残して問題ありません。

### 4. 補助ドキュメント

- `README.md`
- `docs/20260312_確認共有.md`
- `docs/CLAUDE.md`
- `docs/制作指示書_Phase7_positionMode再定義と座標記録.docx`
- `docs/資料(処理済)/...`

経緯や仕様の把握には有用です。ただし GitHub にどこまで載せるかは別途判断できます。

### 5. 生成物

- `bin/`
- `obj/`
- `publish/`
- `installer_output/`
- `artifacts/`

これらは再生成可能です。コード管理の主対象ではありません。

## 推奨する見方

### GitHub の核

- ソースコード本体
- ビルド・配布設定
- 必要最小限の説明文書

### GitHub に載せるか後で決めるもの

- 過去の仕様書
- 確認共有メモ
- docx 資料

### GitHub には基本載せないもの

- 実行結果として生成されるファイル
- 配布用 exe / zip
- ビルド中間生成物

## 次の整理候補

1. `bin/ obj/ publish/ installer_output/ artifacts/` を生成物として明確化
2. GitHub に残す文書を `README.md` と必要最小限へ絞る
3. `.gitignore` を追加する
4. 必要なら `docs/` 内を「残す資料」と「保管資料」に分ける
