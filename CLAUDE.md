## 1. プロジェクト概要

### StartFormとは
「自分が決めた作業環境を、いつでもワンアクションで整える」Windowsデスクトップアプリ。

利用者がプロファイル（作業環境の設計図）を作成し、実行すると、指定したアプリが起動し、指定した画面位置に配置される。

### 便利アプリシリーズ第二弾
第一弾「ReDesktop Resume」（PC再起動後の復元）の姉妹アプリ。StartFormは「仕切り直し型」で、利用者が任意のタイミングで実行する。

## 4. 核心技術（実機テスト済み）

### 起動と配置の融合
Process.Start でアプリを起動し、MoveWindow / SetWindowPos で指定位置に配置する。これがStartFormの核心。

### 比率指定の配置
画面を比率で分割する（合計10）。ピクセル座標ではなく比率を使うことで、モニター解像度が変わっても設定が壊れない。
- 例：画面① = 4:6 → 左40%にExcel、右60%にChrome

### シャドウ補正
Windows 10/11のウィンドウにはシャドウ（影）がある。DwmGetWindowAttribute（DWMWA_EXTENDED_FRAME_BOUNDS、属性値9）でシャドウを除いた描画領域を取得する。
- シャドウ幅：左7px、右7px、下7px（実機テストで確認済み）

### マルチモニター対応
Screen.AllScreens で各モニターの解像度と位置を取得する。3モニター構成で確認済み。

## 6. データ設計

### プロファイルJSON（保存先：%AppData%/StartForm/profiles/）

主要フィールド：
- `profileName` (string) - プロファイル名
- `entries` (array) - アプリ一覧
  - `processPath` (string) - 実行ファイルのフルパス
  - `processName` (string) - プロセス名
  - `filePath` (string?) - 開くファイルのパスまたはURL
  - `filePaths` (string[]?) - ブラウザ用：複数URLの配列
  - `monitor` (int?) - 配置先モニター番号（1始まり、nullは位置指定なし）
  - `ratio` (int?) - 比率値（合計10の中の割り当て）
  - `order` (int?) - 同一モニター内の左からの順番（1始まり）
  - `positionMode` (string) - "ratio" または "none"

### 前回位置記録JSON（保存先：%AppData%/StartForm/last_positions/）

主要フィールド：
- `processName`, `processPath` - プロセス識別
- `posX`, `posY`, `width`, `height` - 位置とサイズ
- `windowState` - Normal / Maximized / Minimized

### インストールレポートJSON（保存先：アプリフォルダ内）

StartFormが追加インストールした機能を記録。アンインストール時に参照する。

## 7. 実行フロー

プロファイル実行時の処理順序：

1. **既起動判定** - プロセス一覧を走査し、既に起動しているか確認
2. **起動処理** - 未起動のアプリを Process.Start で起動（ファイル指定があれば引数に渡す）
3. **ウィンドウ待機** - WaitForInputIdle + Thread.Sleep でウィンドウ生成を待つ
4. **位置配置** - 位置指定ありのアプリを比率→座標変換してMoveWindow/SetWindowPosで配置（シャドウ補正適用）
5. **位置なし配置** - 位置指定なしのアプリを前回記録位置に配置（記録がある場合）
6. **Z順序整列** - 全ウィンドウを設定順に前面に整列
7. **位置記録** - 実行結果の全ウィンドウ位置を記録

## 8. ReDesktop Resume技術資産の活用

以下はReDesktop Resumeからの技術資産。StartFormで転用・拡張する。

| 資産 | StartFormでの活用 |
|------|------------------|
| ScreenCapturer.CaptureAll() | 「現在を拾う」機能の基盤 |
| WindowInfo | プロファイルエントリのベース構造（フィールド追加あり） |
| WindowRestorer.Restore() | プロファイル実行エンジンの基盤（比率→座標変換を追加） |
| ApplyWindowPosition() | シャドウ補正付きウィンドウ配置 |
| BrowserUrlCapturer | ブラウザURL取得（そのまま流用） |
| CommandLineParser | ファイルパス解決（そのまま流用） |
| AppSettings / JSON読み書き | プロファイル管理に拡張 |
| Mutex制御 | 多重起動防止（そのまま流用） |
| Inno Setup (setup.iss) | インストーラーテンプレート |
| ExcludedProcesses | 除外リスト（UWPイマーシブアプリを追加） |

## 9. ファイル取得の優先順位

| 順位 | 手段 | 取得できるもの |
|------|------|---------------|
| 1 | コマンドライン（WMI） | フルパス（ダブルクリック起動時） |
| 2 | COM経由（Shell.Application） | エクスプローラーのフォルダパス |
| 3 | UI Automation | ブラウザのアクティブタブURL |
| 4 | ウィンドウタイトル | ファイル名（パスなし） |
| 5 | 利用者が手動で指定 | 何でも |

## 10. 確認されている壁（採用しないもの）

- **Word COMオブジェクト接続** - GetActiveObject("Word.Application")は64bit/32bit両方で失敗。採用しない。
- **UWPアプリ（Windows設定）** - MainWindowTitleが空。除外対象。
- **ChromeタブのUI Automation一覧取得** - アクティブタブのみ。全タブは取得不可。利用者がURL指定する設計。
